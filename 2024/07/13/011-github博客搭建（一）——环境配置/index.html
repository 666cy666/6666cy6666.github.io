原先准备是在云服务器上搭个动态博客的，后面想想动态的无非多了个统计还有动态修改功能，我个人不是很在意这些，反正在本地写已经习惯了，大不了修改了再上传呗，多大点事。

环境准备

  • Node.js：JavaScript 运行时环境，用于构建和运行博客相关的应用程序。
  • Git：版本控制工具，用来上传和管理博客文件。
  • Typora：Markdown 编辑器，用于书写和编辑博客内容。
  • 一个域名：为你的博客提供一个独立的访问地址。
  • Python 环境：用于批量处理 Markdown 图片格式。如果没有 Python 环境，后续的步骤中我会提供一个可执行文件（exe），但不同主题模板可能需要自行调试。
  • CDN 加速（Cloudflare）：由于 GitHub 的服务器位于国外，直接访问可能会比较慢。我们可以使用 Cloudflare 提供的免费 CDN 服务来加速博客的访问速度。前往 Cloudflare 官网注册账号并添加你的域名，按照提示完成设置

1. 查看所有 Git 配置

要查看包括系统级别在内的所有配置，可以使用：

git config --list

这将展示所有层面上的 Git 配置，包括全局、系统、和本地。

2. 查看特定配置项

如果你只想查看特定的配置项，如用户名，可以使用：

git config user.name

或者查看与远程仓库相关的 URL：

git config remote.origin.url

这些命令分别显示了用户名和当前仓库的默认远程仓库 URL。

3. 更改 Git 配置

如果你需要修改某项配置，例如改变用户名或邮箱，可以使用：

git config --global user.name "新用户名"
git config --global user.email "新邮箱地址"

与github关联后可以使用上面的指令查看是否成功，若看到账号邮箱与注册好的github账号关联成功，恭喜你，已经成功5%了！
